# AGENTS.md

## Project instructions
This repository is a prototype for a product photo enhancement app.

Always read SPEC.md before making changes.

## Rules
- Do not add features outside the MVP scope.
- Prefer small, reviewable changes.
- Keep the architecture simple and replaceable.
- Do not introduce heavy dependencies unless clearly necessary.
- If image processing is not implemented yet, use a placeholder pipeline that can be swapped later.
- Explain what you changed in simple terms.

## Coding style
- Use clear names.
- Avoid overengineering.
- Separate UI code and processing code when possible.

## Workflow
- First understand the current goal from SPEC.md.
- Then propose the smallest next step.
- Then implement only that step.

## When unsure
- Do not guess large product decisions.
- Leave a short note in comments or summary if something is intentionally stubbed.
